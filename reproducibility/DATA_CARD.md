## Data origin and generation

**All data is fully synthetic.** No human subjects. No real OTA inventory.

- **Market scenarios**: Generated by `generate_small_market` (in `src/tourmart/scenarios.py`) with fixed seed=42. Deterministic; re-runnable without GPU.
- **Phase 1c raw.jsonl**: Generated by running Qwen-14B-AWQ and Llama-3.1-8B as LLM readers on Server B (NVIDIA RTX 3090 24GB, CUDA 12.1, PyTorch 2.4.0+cu121) using vLLM 0.6+. These require GPU to re-generate. Wall times: ~13 min (Qwen) + ~20 min (Llama) + ~51 min (Phase 1c diagnostic replay). Pre-computed outputs are shipped so reviewers can skip GPU re-generation.
- **Permutation null**: Generated by `reproduce_permutation.py` (CPU, ~2-5 min) from the raw.jsonl files above. Deterministic at seed=12345.

## Schema: raw.jsonl rows (Phase 1c)

Each row is a JSON object with at minimum:
```json
{
  "scenario_id": "...",
  "traveler_id": "...",
  "bundle_id": "...",
  "variant": "original" | "factual",
  "signal_wt": 0.0–1.0,
  "baseline_surplus": float,
  "budget": float,
  "tau": float,
  "features": {
    "perceived_fit_delta": float,
    "perceived_risk": float,
    "trust_score": float,
    "urgency_felt": float
  },
  "distance_to_threshold_frac": float
}
```
The `episode_seed` column is present only in the `.with_episode_seed.raw.jsonl` variants (reconstructed by `add_episode_seed.py`). See paper §L197 footnote and `add_episode_seed.py` for disambiguation methodology.

## Pairing note (paper-critical)

The paper's headline n=143 uses **3-tuple pairing** `(scenario_id, traveler_id, bundle_id)`. A 5-tuple pairing `(scenario_id, traveler_id, bundle_id, signal_wt, episode_seed)` yields n=409 and is NOT the published analysis. `reproduce_permutation.py` implements the 3-tuple key explicitly. See script docstring for details.

## Six-gate audit outputs (in `../../../results/`)

Seven `msgcap_v7_*.md` files document the six-gate quality audit on OTA messaging arms. See `gate_history.md` for threshold history and rationale. These are read-only reference documents; no script in this package reads them directly.
